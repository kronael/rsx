**Lesson (the hard way):** my rapid doc commits raced the tui sub's shared git
index — `git add X && git commit` swept the sub's staged deletions into a docs
commit (77e2961). Nothing broke (all wanted changes) but the history is
mislabeled and unfixable without amend/rebase (forbidden). Rule: **never
`git commit` while a code sub commits on the same tree**; if unavoidable, use
an explicit-pathspec commit (`git commit <file>`, only-mode) so it can't sweep
the index. Holding BUGS.md (fable + codex findings) + the publish roadmap
(book → matching → gateway+marketdata → tui → hosting/install-run → blog)
until the tui-fix sub finishes.

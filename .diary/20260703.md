## 10:09 2026-07-03 — rsx-tui: developed against tests

Founder: the TUI must be developed against tests, with integration + play-like
tests like the rest of the repo. Restructured rsx-tui from a single bin into
lib+bin so the UI core is drivable headless:
- `rsx_tui` lib: `App` (pub fields), `Side`, `Control{Continue,Quit}`, `draw`,
  `handle_key(&mut App, KeyCode) -> Control`, `App::spread()`.
- `main.rs`: thin — `ratatui::init` + poll/read loop calling `handle_key`.

Tests use ratatui `TestBackend` (render into an off-screen `Buffer`, assert on
cell symbols) — the TUI analog of Playwright driving a browser:
- `tests/render_test.rs` (5): symbol + DEMO badge, book/spread/positions/
  trades panels, BBO levels visible, tiny-terminal no-panic.
- `tests/play_test.rs` (3): scripted session — press s→Sell, b→Buy, x no-op,
  q→Quit, Esc→Quit; panels persist across the session.

One fix during TDD: the spread cell is width-clamped (7-col table cell cut
"spread 1"→"spread "), so I extracted `App::spread()` and test the computation
directly + assert the label renders — don't assert width-truncated values.

`cargo test -p rsx-tui` 8/0; clippy clean. Makefile test tiers use
`--workspace --tests --lib`, so these run under `make test` + `make e2e` with
no extra wiring.

## Q&A this session

- **QUIC lifecycle** (why BENCH-QUINN-ACCEPT-BI panics): QUIC nests a
  connection lifecycle (TLS-1.3 handshake in-transport → Connection → close/
  idle-timeout) and a stream lifecycle. `open_bi()` returns the stream pair
  LOCALLY and sends nothing on the wire; the peer's `accept_bi()` only resolves
  once the opener WRITES (first STREAM frame). The bench accepts before any
  write → unwrap panics. Fix: client writes a priming byte before server
  accept.
- **TUI is Rust/ratatui, not Go.** Deciding factor: reuse rsx-types/rsx-messages
  wire types directly (one source of truth) vs a Go/Bubbletea client that
  re-declares every wire struct + webproto framing. Also one workspace/toolchain
  and ratatui's TestBackend gives deterministic UI tests.

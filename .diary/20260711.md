### 12:40 ‚Äî all 4 reports in + fable MVP plan complete (`.ship/45-ARIZUKO-LLM/PLAN.md`)

Findings (each `research-{A..D}.md` cites file:line):
- **A deploy**: Docker-per-turn is **hard-mandatory** ‚Äî `runed` only wires
  `NewDockerRuntime`; the sole non-Docker path is a test-only `FakeRuntime`, no
  dev/exec flag. Min daemons = `authd+routd+runed` (+`webd` for HTTP). Each turn
  spawns a fresh `arizuko-ant:latest` (Node22+Chromium+`@anthropic-ai/claude-code`).
  Zero-config CLI ingress: `arizuko send <inst> <folder> "msg" --wait` (operator
  DB authority, no token). CLI verified: `create/run/group/send/token issue chat/
  secret/network/apply`.
- **B contract**: HTTP ingress `POST /chat/{token}` body `{content, topic}`; route
  token minted once via routd `POST /v1/route_tokens/chat` (open locally, no
  `AUTHD_URL`); egress SSE (`Accept: text/event-stream`) or `?wait=<sec>`.
  Threads = client-chosen `topic`; `sessions` PK `(group_folder, topic)`.
- **C runtime/tools**: RSX has **no REST API** (WS+protobuf) ‚Üí cleanest live-data
  path is an arizuko **MCP-subprocess connector** (`[[mcp_connector]]` in
  `connectors.toml` ‚Üí a small host-side binary routd spawns; reaches RSX over
  localhost, no container egress). Zero arizuko-core change. Runtime pinned to
  Claude Agent SDK; creds via `ANTHROPIC_API_KEY`.
- **D auth/social**: min local auth trivial (open routd w/o `AUTHD_URL`; else
  service-token). Social attach = **one route row**
  (`arizuko group rsx add bluesky:user/<did> main`); bskyd self-registers.
  Onboarding skippable.

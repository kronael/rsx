## 21:00

Discovery (from the in-process matching round-trip): the ME
algorithm is **320 ns p50** — dedup 70 + wal_accept 60 + match 80
+ wal_events 110. The whole 10us round-trip is plumbing:
CmpSender::send body ×2 = 7.8us (sendto syscall floor),
UDP-loopback + try_recv ×2 = 1.3us.

So production 1128us p50 has ~110× overhead vs the floor.
All in CMP plumbing + tokio scheduling + the 100us sleeps.

## 21:30

Send-path analysis: CmpSender::send body is 3.9us p50.
Breakdown (estimated, not directly measured):
  - sendto syscall (loopback UDP): ~1.5-2us  ← floor
  - crc32fast SIMD over 128B: ~30ns
  - WalHeader build + 2 memcpy: ~50ns
  - NAK ring slot copy 144B: ~10ns
  - measurement overhead: ~1us

Production gateway uses monoio (io_uring), so sendto goes
through the submission ring — likely ~half cost. Bench
overstates production by ~1-2us on the send legs.

Strategic fix to kill the 100us sleep: refactor
rsx-dxs::CmpReceiver to use monoio::net::UdpSocket and
expose `readable().await`. Gateway main loop then does
`monoio::select!` on socket-readable + maintenance tick.
~50 LOC. Out of /fin scope.
